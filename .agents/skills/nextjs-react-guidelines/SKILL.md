# Next.js and React Guidelines

Frontend rules:
- Follow existing component, hook, and route conventions.
- Keep components focused and avoid bloated page files.
- Preserve existing state management and data-fetching patterns unless there is a clear reason to change.
- Validate and sanitize client inputs appropriately.
- Avoid exposing sensitive implementation details in the browser.
- Keep naming consistent across components, props, hooks, and tests.

*(Applicable to: React frontend files in app, src, components, pages directories)*
